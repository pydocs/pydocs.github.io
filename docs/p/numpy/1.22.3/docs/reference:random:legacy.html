Legacy Random Generation

The :None:None:`RandomState` provides access to legacy generators. This generator is considered frozen and will have no further improvements. It is guaranteed to produce the same values as the final point release of NumPy v1.16. These all depend on Box-Muller normals or inverse CDF exponentials or gammas. This class should only be used if it is essential to have randoms that are identical to what would have been produced by previous versions of NumPy.

:None:None:`RandomState` adds additional information to the state which is required when using Box-Muller normals since these are produced in pairs. It is important to use :None:None:`RandomState.get_state`, and not the underlying bit generators :None:None:`state`, when accessing the state so that these extra values are saved.

Although we provide the :None:None:`MT19937` BitGenerator for use independent of :None:None:`RandomState`, note that its default seeding uses :None:None:`SeedSequence` rather than the legacy seeding algorithm. :None:None:`RandomState` will use the legacy seeding algorithm. The methods to use the legacy seeding algorithm are currently private as the main reason to use them is just to implement :None:None:`RandomState`. However, one can reset the state of :None:None:`MT19937` using the state of the :None:None:`RandomState`:

.. code-block:: ['python'] 
from numpy.random import MT19937 from numpy.random import RandomState rs = RandomState ( 12345) mt19937 = MT19937 ( ) mt19937.state = rs.get_state ( ) rs2 = RandomState ( mt19937) # Same output rs.standard_normal ( ) rs2.standard_normal ( ) rs.random ( ) rs2.random ( ) rs.standard_exponential ( ) rs2.standard_exponential ( )

Functions in `numpy.random`

Many of the RandomState methods above are exported as functions in numpy.random This usage is discouraged, as it is implemented via a global :None:None:`RandomState` instance which is not advised on two counts:

  • It uses global state, which means results will change as the code changes

  • It uses a :None:None:`RandomState` rather than the more modern :None:None:`Generator`.

For backward compatible legacy reasons, we cannot change this. See random-quick-start .
