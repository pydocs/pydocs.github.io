Feature Flags

Read the Docs offers some additional flag settings which are disabled by default for every project and can only be enabled by :None:None:`contacting us through our support form` or reaching out to the administrator of your service.

            <Unimplemented 'target' '.. _contacting us through our support form: https://docs.readthedocs.io/en/stable/support.html'>

Available Flags

PIP_ALWAYS_UPGRADE : :None:featureflags:`PIP_ALWAYS_UPGRADE`

UPDATE_CONDA_STARTUP : :None:featureflags:`UPDATE_CONDA_STARTUP`

The version of conda used in the build process could not be the latest one. This is because we use Miniconda, which its release process is a little more slow than conda itself. In case you prefer to use the latest conda version available, this is the flag you need.

CONDA_APPEND_CORE_REQUIREMENTS : :None:featureflags:`CONDA_APPEND_CORE_REQUIREMENTS`

Makes Read the Docs to install all the requirements at once on conda create step. This helps users to pin dependencies on conda and to improve build time.

DONT_OVERWRITE_SPHINX_CONTEXT : :None:featureflags:`DONT_OVERWRITE_SPHINX_CONTEXT`

LIST_PACKAGES_INSTALLED_ENV : :None:featureflags:`LIST_PACKAGES_INSTALLED_ENV`

DONT_CREATE_INDEX : :None:featureflags:`DONT_CREATE_INDEX`

When Read the Docs detects that your project doesn't have an index.md or README.rst , it auto-generate one for you with instructions about how to proceed.

In case you are using a static HTML page as index or an generated index from code, this behavior could be a problem. With this feature flag you can disable that.
