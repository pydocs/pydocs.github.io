multinomial_gen(seed=None)

A multinomial random variable.

Parameters

x : array_like

Quantiles, with the last axis of x denoting the components.

%(_doc_default_callparams)s :
%(_doc_random_state)s :

Methods

Notes

%(_doc_callparams_note)s

Alternatively, the object may be called (as a function) to fix the n and p parameters, returning a "frozen" multinomial random variable:

The probability mass function for :None:None:`multinomial` is

.. math:: ['']

f(x) = \frac{n!}{x_1! \cdots x_k!} p_1^{x_1} \cdots p_k^{x_k},

supported on :None:math:`x=(x_1, \ldots, x_k)` where each :None:math:`x_i` is a nonnegative integer and their sum is :None:math:`n`.

.. versionadded:: [' 0.19.0']

scipy.stats._multivariate.multinomial_gen

Examples

>>> from scipy.stats import multinomial
... 
... rv = multinomial(8, [0.3, 0.2, 0.5])
... rv.pmf([1, 3, 4])
0.042000000000000072

The multinomial distribution for :None:math:`k=2` is identical to the corresponding binomial distribution (tiny numerical differences notwithstanding):

>>> from scipy.stats import binom
... 
... multinomial.pmf([3, 4], n=7, p=[0.4, 0.6])
0.29030399999999973
>>> binom.pmf(3, 7, 0.4)
0.29030400000000012

The functions pmf , logpmf , entropy , and cov support broadcasting, under the convention that the vector parameters ( x and p ) are interpreted as if each row along the last axis is a single object. For instance:

>>> multinomial.pmf([[3, 4], [3, 5]], n=[7, 8], p=[0.3, 0.7])
array([0.2268945,  0.25412184])

Here, x.shape == (2, 2) , n.shape == (2,) , and p.shape == (2,) , but following the rules mentioned above they behave as if the rows [3, 4] and [3, 5] in x and [.3, .7] in p were a single object, and as if we had x.shape = (2,) , n.shape = (2,) , and p.shape = () . To obtain the individual elements without broadcasting, we would do this:

>>> multinomial.pmf([3, 4], n=7, p=[0.3, 0.7])
0.2268945
>>> multinomial.pmf([3, 5], 8, p=[0.3, 0.7])
0.25412184

This broadcasting also works for cov , where the output objects are square matrices of size p.shape[-1] . For example:

>>> multinomial.cov([4, 5], [[0.3, 0.7], [0.4, 0.6]])
array([[[ 0.84, -0.84],
        [-0.84,  0.84]],
       [[ 1.2 , -1.2 ],
        [-1.2 ,  1.2 ]]])

In this example, n.shape == (2,) and p.shape == (2, 2) , and following the rules above, these broadcast as if p.shape == (2,) . Thus the result should also be of shape (2,) , but since each output is a :None:math:`2 \times 2` matrix, the result in fact has shape (2, 2, 2) , where result[0] is equal to multinomial.cov(n=4, p=[.3, .7]) and result[1] is equal to multinomial.cov(n=5, p=[.4, .6]) .
